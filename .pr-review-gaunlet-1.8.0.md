## BLOCKING ISSUES

### [B1] CRITICAL: Non-deterministic hash in pre-commit gate
**File:** `plugins/gauntlet/hooks/precommit_gate.py:132`
**Severity:** Critical (breaks core functionality)

Python's `hash()` uses random seed per process (PYTHONHASHSEED).
The pass-token mechanism compares hashes across separate hook
invocations, so tokens will almost never match:

```python
# Current (broken across processes):
return str(hash(result.stdout.strip()))
```

Verified: `hash("test")` returns different values across `python3`
invocations. Tests pass because they mock `_get_staged_hash`.

**Fix:** Use `hashlib.sha256(...).hexdigest()[:12]` for deterministic
hashing.

---

### [B2] HIGH: Test race condition in oracle daemon tests
**File:** `plugins/oracle/tests/unit/test_daemon.py:115-117`
**Severity:** High (flaky CI)

Fixture waits for port file to exist but not for content to be
written. Verified: running oracle tests fails with:

```
ValueError: invalid literal for int() with base 10: ''
```

The port file exists but is empty when the test reads it.

**Fix:** Wait for file to have non-empty content, not just existence.
