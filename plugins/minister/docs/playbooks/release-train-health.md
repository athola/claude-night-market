# Release Train Health Playbook

## Gate 1 – Scope Freeze

- validate every blocking issue/PR is linked in the tracker data.
- Apply `release:train-X` label on GitHub issues for traceability.
- Stakeholders sign off via comment generated by `release-health-gates` skill.

## Gate 2 – Quality Signals

| Signal | Source | Action |
|--------|--------|--------|
| CI Checks | GitHub required checks list | Must be green for 24h or rerun with justification. |
| Docs | `docs-needed` label cleared | Confirm doc PR merged or tracked in follow-up issue. |
| Support Readiness | Triage discussion | Document known issues + mitigations. |

## Gate 3 – Communication

1. Generate release note skeleton inside deployment PR description.
2. Attach Status Template excerpt focusing on Executed vs Upcoming tasks.
3. Update Projects view `Release Train` with final burndown metrics.

## Gate 4 – Post-Ship Review

- Re-run Initiative Pulse to confirm tasks moved to `Done`.
- Export CSV snapshot and archive under `/docs/artifacts/` (create as needed).
- Log issues discovered during bake time as `post-release` label for visibility.
